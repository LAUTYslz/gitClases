Areas de GIT

Working Directory

El Area trabajo corresponde a la carpeta donde creamos el repositorio y basicamente vamos a crear, editar o eliminar archivos o subcarpetas. En muy pocas ocaciones utilizaremos la consola y solamente necesitamos implementar el comando GIT que inicializa el repositorio.

Staging Area (INDEX)

El area de control de cambios corresponde a un archivo adentro de la carpeta .git que gestiona a que archivos le realizamos cambios. Tambien se encarga de realizar las capturas de codigo (SNAPSHOT) para su posterior confirmacion cuando hayamos finalizado con el trabajo correspondiente.

Repository (local/remote)

El almacen de cambios corresponde a una estructura de carpetas dentro de la carpeta .git que contiene las confirmaciones realizadas en formato BLOB (Binary Large Object). Se utilizan como referencia para comparar, visualizar y modificar el contenid de nuestro proyecto.
